# 微录 MVP 实施计划

## 0. 总览

- 周期：1 个完整会话
- 任务数：10 组 / 41 个（来自 `openspec/changes/init-weilu-mvp/tasks.md`）
- 工作方式：用户选定（默认推荐：branch + executing-plans）
- 提交粒度：每个 Task Group 至少 1 个提交，跨 group 拆 commit

## 1. 提交与目录约定

- 主分支：main
- 开发分支：`feat/init-weilu-mvp`（由用户选 branch 模式时创建）
- commit 规范：`<group>: <task-id> <一句话描述>`
  - 例：`data-layer: 1.1 update miniprogram app.json pages`
- 不引入 prettier / eslint 自动化（仓库无现成配置），按 Taro3 习惯手写格式

## 2. 实施分组与顺序

依赖关系：
```
1 (脚手架)
  └→ 2 (数据层)
       ├→ 3 (人物)
       ├→ 4 (事件)
       └→ 5 (物品)
            └→ 6 (关联)
                 └→ 7 (录入)
                      ├→ 8 (提醒)
                      └→ 9 (周复盘)
                           └→ 10 (联调发布)
```

## 4. 风险与回退

- **关联 N+1**：若详情页云函数超时，回退方案：分两次 callFunction（先 entity 后 relations）
- **订阅模板未申请**：所有 reminder 走 in_app，UI 提示「请在 MP 平台申请模板」
- **真机调试受限**：本地开发者工具 + 一台真机即可
